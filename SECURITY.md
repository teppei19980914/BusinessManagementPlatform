# セキュリティポリシー

- 作成日: 2026-04-22
- 対象: 本リポジトリ (たすきば Knowledge Relay) の全ソースコード / 本番稼働サービス

本プロジェクトはセキュリティを重視しており、脆弱性の発見・報告に感謝いたします。
以下のガイドラインに沿ってご報告ください。

---

## 1. 脆弱性の報告方法

**「どこからでも受け入れる」方針** です。以下のいずれの経路でもご報告いただけます。
可能な限り **非公開チャネル** (1 または 2) をご利用ください。

### 推奨: 1) GitHub Security Advisory (最優先)

本リポジトリの **[Security タブ](https://github.com/teppei19980914/BusinessManagementPlatform/security) → "Report a vulnerability"**
から非公開で報告できます。GitHub のネイティブ機能で、メールアドレス開示不要・
添付ファイル対応・議論ログが残ります。

### 2) メール

非公開メールでのご報告も受け付けます:

- **連絡先**: teppei09141998@gmail.com

件名の先頭に `[SECURITY]` をお付けいただけると検索しやすくなります。

### 3) GitHub Issues (軽微な内容のみ)

**明らかに悪用リスクのない情報** (例: セキュリティ機能の UX 改善提案、
セキュリティヘッダの設定提案等) に限り、通常の
[GitHub Issues](https://github.com/teppei19980914/BusinessManagementPlatform/issues)
でも受け付けます。

> ⚠️ **実際に悪用可能な脆弱性 (認可バイパス / XSS / SQL Injection 等) は
> 絶対に公開 Issue に投稿しないでください**。サービス利用者に影響が及びます。
> 上記 1 または 2 の非公開チャネルをご利用ください。

---

## 2. ご報告いただきたい情報

以下の情報があると対応がスムーズになります (全てが揃わなくても受け付けます):

- 脆弱性の種類 (例: XSS, 認可バイパス, SQL Injection)
- 影響範囲 (どのユーザ / どの操作が影響を受けるか)
- 再現手順 (ステップごと + 環境情報)
- 想定される影響 (データ漏洩 / 権限昇格 / サービス停止等)
- 可能であれば修正案

---

## 3. 対応方針と目標時間

単独開発・招待制サービスという規模・特性を踏まえ、現実的な目標値を設定しています。
サービス規模拡大に応じて、より厳しい値に見直します。

| 深刻度 | 初回応答 | 修正目標 | 例 |
|---|---|---|---|
| **Critical** | 48 時間以内 | 1 週間以内 | 認証バイパス / admin 権限奪取 / 生データ流出 |
| **High**     | 1 週間以内  | 1 ヶ月以内 | 横断的 IDOR / 認可境界の穴 / XSS |
| **Medium**   | 2 週間以内 | 3 ヶ月以内 | 情報露出 (影響範囲限定的) / CSRF 弱点 |
| **Low**      | 2 週間以内 | 次回リリースサイクル | 設定最適化 / 監査ログ改善提案 |

> ※ 週末・祝日・業務都合で若干遅延する場合があります。48 時間以内の初動が
> 困難な状況では、受付確認だけ先行してお送りします。

## 4. サポート対象バージョン

| バージョン | サポート |
|---|---|
| `main` ブランチの最新コミット | ✅ |
| 過去 30 日以内のリリース | ✅ (タグ付け後追加予定) |
| それ以前のバージョン / 独自 fork | ❌ |

本リポジトリはプレリリース段階 (2026-05-01 公開予定) のため、現時点では
`main` ブランチのみをサポート対象としています。正式リリース (2026-06-01) 以降は
リリース tag 基準でのサポート範囲を明示します。

---

## 5. 対応対象外の事項

以下は脆弱性として受付対象外とさせていただきます:

- **既知の設計上の制約** (e.g., 招待制のため外部ログインフォームが無い等、
  [docs/DESIGN.md](./docs/DESIGN.md) に記載済みの設計判断)
- **サードパーティ依存の脆弱性 (自動検知済)**
  - `pnpm audit` / Dependabot / CodeQL で検知されているものは既に対応中
  - ただしそれ以外の依存や、依存関係の**組み合わせによって顕在化する問題**は受付対象
- **セルフホスト環境 (on-prem) 固有の設定不備** (Phase 2 以降対応)
- **社会工学的攻撃** (phishing 等、コード上の問題ではないもの)

## 6. 開示方針 (Coordinated Disclosure)

責任ある開示 (Coordinated Disclosure) に従います:

1. 報告者と修正計画を共有し、修正リリースまで **非公開** を維持
2. 修正リリース後、報告者の同意を得て **GitHub Security Advisory** で開示
3. **謝辞を公開**: 報告者のお名前 (または匿名希望 OK) を commit メッセージ /
   Security Advisory に記載

公表前の開示 (0-day 公開等) は本プロジェクトの利用者に影響が及ぶため、
事前調整をお願いいたします。
