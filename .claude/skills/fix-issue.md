# 問題修正スキル

## 手順

1. 問題のレポートや指摘事項を確認する
2. **情報源の選定（必須）** — 調査が必要な場合は CLAUDE.md「情報源の信頼性ルール」に従い、公式ドキュメント・公式リポジトリを最優先で参照する。推測ベースの実装は禁止
3. **設計影響の判断** — 新しい入力経路・権限境界・外部連携が発生する場合は、修正前に `/threat-model` を実行
4. 該当するソースコードを特定・修正する
5. テストコードを必ず追加・修正する（**セキュリティテストも含む**）
6. **観点別セキュリティレビュー** — 以下を並列実行（Task tool で同時起動）
   - `auth-reviewer` — 認証/認可/IDOR
   - `injection-reviewer` — SQL/コマンド/SSRF/Path
   - `xss-reviewer` — XSS/CSP/CSRF
   - `secret-reviewer` — 機密情報/ログ漏洩
   - `dependency-reviewer` — 既知脆弱性（依存追加時のみ）
   - `performance-reviewer` — N+1/再描画/並列化
7. 指摘された HIGH 以上の問題は全て修正する
8. コミット前チェックを実施（以下すべて）

## 横展開チェック

- 修正した問題と同じパターンが他ファイルにないか検索
- テストコード内で旧文言を検索し全て新文言に修正
- 共通処理の変更は関連する全ファイルに横展開

## セキュリティチェック（エージェントで自動化される部分以外）

- **多層防御は各エージェントが担当** — 本スキルでは「修正漏れがないか」を確認
- **セキュリティテスト**: 以下を必ず追加
  - 認可境界テスト（他ユーザーのリソースアクセス試行）
  - 不正入力テスト（SQLi/XSS payload, 巨大入力, Unicode正規化）
  - 認証バイパス試行
- **新規依存を追加した場合**: `dependency-reviewer` を必ず実行

## パフォーマンスチェック

- ループ内DB問い合わせ禁止（一括取得→メモリ上フィルタ）
- 描画オブジェクトのキャッシュ
- 不要な再描画の抑制
- 独立した非同期操作の並列化

## デプロイチェック

ローカルで以下を実行し全て成功を確認:

```bash
pnpm lint
pnpm test
pnpm build
```

## ドキュメント更新

変更内容に応じて該当ドキュメントを更新:
- 仕様変更 → `SPECIFICATION.md`
- 設計変更 → `DESIGN.md`（セキュリティ設計セクション含む）
- 要件追加 → `REQUIREMENTS.md`
- 運用手順変更 → `OPERATIONS.md`
