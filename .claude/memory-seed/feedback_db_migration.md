テーブルにカラムを追加する際は、マイグレーション定義を必ずセットで行うこと。

**Why:** GrowthEngine で Dream テーブルに sortOrder を追加した際にマイグレーション未定義で、WASM SQLite でスキーマ不一致によりDBが再作成されユーザーデータが全消失した。クラウドバックアップから復元できたが重大インシデント。

**How to apply:**
- カラム追加は「テーブル定義・マイグレーション・モデル・サービス・エクスポート」の5点セットで変更
- ORM の自動マイグレーション機能に頼らず、明示的にバージョン管理する
- DB変更時はテスト環境で「既存DBからのアップグレード」を必ず検証
