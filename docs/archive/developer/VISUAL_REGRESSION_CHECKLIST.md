# 視覚回帰チェックリスト (VISUAL_REGRESSION_CHECKLIST.md)

> 本書は **PR #76 の Tailwind 色 → semantic token 移行** および **PR #78 の color-scheme 追加**
> 等の UI 横断的な変更後に、10 テーマ × 主要画面で視覚崩れがないことを目視確認するための
> チェックリストです。
>
> 自動テストでは検出できないため、UI 変更を伴う PR のマージ前にユーザ自身が実施します。

---

## 前提

- Vercel Preview Deployment URL を取得済 (PR ごとに自動発行)
- ログイン済 (テスト用アカウントで OK)
- ブラウザ: Chrome / Edge / Safari いずれか (主要ブラウザの最新版)

---

## チェック手順

### 1. 全テーマで切替確認

設定画面 → 画面テーマ で以下 10 種を順に切り替え、各テーマで「§2 画面別チェック」を実施:

- [ ] light (ライトテーマ・デフォルト)
- [ ] dark (ダークテーマ)
- [ ] pastel-blue (パステル青)
- [ ] pastel-green (パステル緑)
- [ ] pastel-yellow (パステル黄)
- [ ] pastel-red (パステル赤)
- [ ] pop-blue (ポップ青)
- [ ] pop-green (ポップ緑)
- [ ] pop-yellow (ポップ黄)
- [ ] pop-red (ポップ赤)

> **時短Tips**: dark + light + pastel-blue + pop-red の 4 種で代表確認 → 問題なければ全テーマ OK と判定。
> 問題発見時は他テーマも確認。

### 2. 画面別チェック

各テーマで以下の画面を順に開き、表示崩れがないことを確認:

#### 2.1 ログイン前

- [ ] **/login**: フォーム入力欄が見える / 「ログイン」ボタンが押せる / リンク文字が読める
- [ ] **/reset-password**: 同上

#### 2.2 ダッシュボード基本

- [ ] **/projects**: 一覧テーブル背景・罫線・テキストが読める / フィルタ UI が機能する
- [ ] **/projects/[id]** (詳細): 概要タブの「開発方式」「目的」「背景」等が読める
- [ ] **/projects/[id]?tab=tasks** (WBS): タスクツリーが読める / 状態バッジに違和感なし
- [ ] **/projects/[id]?tab=gantt** (ガント): バー / 罫線 / マイルストーンダイヤが見える
- [ ] **/projects/[id]?tab=risks** (リスク): 一覧の優先度・状態バッジが読める
- [ ] **/projects/[id]?tab=retrospectives** (振り返り): 各セクション (KPT) が読める
- [ ] **/projects/[id]?tab=knowledge** (ナレッジ): 行クリックで編集ダイアログが開く

#### 2.3 横断ビュー

- [ ] **/risks** (全リスク): テーブルが読める
- [ ] **/issues** (全課題): 同上
- [ ] **/retrospectives** (全振り返り): 同上
- [ ] **/knowledge** (全ナレッジ): 同上
- [ ] **/all-memos** (全メモ): 同上
- [ ] **/my-tasks** (マイタスク): 個人タスク一覧が読める

#### 2.4 個人設定 / 管理

- [ ] **/settings**: テーマ選択 UI のスウォッチ (色見本) が各テーマで識別可能
- [ ] **/settings**: パスワード変更フォームが読める / MFA カード見える
- [ ] **/admin/users**: 管理者画面のユーザ一覧が読める

### 3. 操作系チェック

各テーマで少なくとも 1 つは以下を実行:

- [ ] **編集ダイアログを開く** (リスク行クリック等): フォームの全フィールドが読める
- [ ] **ネイティブ select を開く**: ドロップダウン内の選択肢が背景と区別できる (PR #78 修正対象)
- [ ] **エラーバナー表示** (パスワード変更で既存と同じ値を入力等): 赤背景・赤文字が読める
- [ ] **成功バナー表示** (テーマ変更成功時): 緑背景・緑文字が読める
- [ ] **削除確認モーダル**: 背面オーバーレイがダイアログを際立たせている
- [ ] **ローディングオーバーレイ**: 操作中の画面暗転 (or 明転) が分かる

### 4. レスポンシブチェック (任意)

- [ ] スマホサイズ (375px 幅) でテーブルが横スクロール可能
- [ ] 画面最大化で過剰な余白がない

## 既知の判定基準

| 状態 | 判定 |
|---|---|
| 文字が背景に溶け込んで読めない | ❌ NG (PR を blocker としてレビュー差し戻し) |
| 色が薄すぎて識別困難だが読めなくはない | 🟡 要相談 (DESIGN.md §29 のテーマ定義値の調整候補) |
| 余白・罫線がない / ずれている | ❌ NG (CSS バグ) |
| ダーク背景に明るい文字が均等に見える | ✅ OK |
| パステル/ポップで色味がそのテーマらしく出ている | ✅ OK |

---

## 問題発見時のアクション

1. **スクリーンショット撮影**: 該当画面 + 該当テーマ + 不具合箇所をマーク
2. **PR コメントで報告** (該当 PR のレビューに添付)
3. **重要度評価**:
   - 重要度 高 (読めない / 操作不能): PR マージを保留
   - 重要度 中 (読みづらい): 別 PR で改修候補として登録
   - 重要度 低 (微妙な見栄え): 既知制約として DESIGN.md §21.4.6 に記録
4. **修正 PR**: テーマ定義値 (`src/config/theme-definitions.ts`) の該当 token を調整、もしくは
   semantic token のマッピングを見直す
